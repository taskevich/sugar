1. Алгоритм добавления и обновления квеста и его фото

    После того, как маршрут .../panel/add вернет id созданного квеста
    нужно отправить запрос с этим id маршруту .../panel/set_photo
    если уж он ничего не вернет, то похуй, значит что-то не то отправили

    Также и на обновление квеста. Как только после обновления инфы вернется
    id и есть фото, то отправляем запрос по .../panel/set_photo

2. Алгоритм фото для квестов

    После добавления фото по маршруту .../add_photo вернется его путь.
    По маршруту можно получить все фото, в зависимости от параметров .../get_photos

    Маршрут .../get_image принимает путь из .../get_photos или .../add_photo вернет файл
    и возвращает файл, поэтому нужно как-нибудь его преобразовать в blob:...
    как это делает fastapi blob:http://45.12.238.117:8000/5343d206-6dee-4a34-b026-e0e3a1cefa71

    На крайний случай, можно просто создать файл или преобразовать в base64, замедлений уже быть не должно

3. Вместо бесконечных запросов, если данные не загрузились, например фотки, написать типо "Не удалось загрузить фото"
Если нет ошибок, но массив фоток пустой, то "нет фото"
